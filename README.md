# LiteCart QA Documentation

This repository is intended to contain manual QA documentation for the LiteCart demo application.

The project is currently in progress. The documentation will be created step by step, starting with basic project analysis, requirements extraction, and manual test design.

## Project Goal

The goal of this repository is to build a structured QA documentation base for LiteCart, including:

* functional requirements;
* manual test cases;
* checklists;
* bug report examples;
* traceability between requirements and tests.

This documentation will later be used as a basis for different test automation implementations.

## Planned Structure

```text
litecart-qa-docs/
│
├── README.md
├── requirements/
├── test-cases/
├── checklists/
├── bug-reports/
├── traceability/
└── reports/
```

## Documentation Language

The main documentation will be written in English.

A short French presentation may be added later to make the project easier to review by French-speaking recruiters or technical interviewers.
